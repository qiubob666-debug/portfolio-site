# RUNBOOK · 标准化操作手册
## 让所有重复操作可重复、可移交、可自动化

---

## 什么是 RUNBOOK

RUNBOOK 是"做这件事的标准化步骤手册"。

任何手动操作发生 3 次以上,**必须**沉淀为 RUNBOOK,否则:
- 第 4 次还得重新想步骤(浪费时间)
- 移交给 AI Agent 做时,Agent 不知道怎么做(浪费上下文)
- 出错时无法定位是哪步错了(浪费调试时间)

## 给 AI Agent 的 RUNBOOK 操作命令

### 执行
- 接到任务时,先查是否有对应 RUNBOOK
- 严格按 RUNBOOK 步骤执行
- 任何偏离都必须报告 Bob

### 创建
- 发现新的重复操作,起草 RUNBOOK 草稿
- 由 Bob 实际执行 1 次后,验证 RUNBOOK 正确性
- 验证通过后,标记为 Validated

### 优化
- 每完成 10 次同一 RUNBOOK 的执行,反思:
  - 哪步可以自动化?
  - 哪步可以合并?
  - 哪步出过错?
- 提出优化建议给 Bob 审阅
