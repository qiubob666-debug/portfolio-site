# ADR-0011: 客群分层与对应落地页策略

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Commercial
**关联 ADR:** 上游 ADR-0001 / 关联 ADR-0021

---

## 背景

bobqiushao.online 主门户面对**多种潜在客群**,但每种客群关心的痛点完全不同:

- 跨境电商卖家:佣金太高 / 平台风险 / 想自建独立站
- 工厂老板:需要正规英文官网 / 想接海外询盘 / Google 搜得到
- 品牌方:多产品线运营 / 多语言 / 数据所有权
- 个人(顾问/KOL/设计师):想要专业线上名片

如果用一个主页"一锅端"试图说服所有人,结果是**谁都说服不了**。

但同时,我们不能为每个客群独立做 4 个完整网站(成本和维护负担过重)。

## 决策

**采用"主门户 + 客群专属落地页"双层架构。**

### 架构

```
bobqiushao.online                  ← 主门户(Bob 是谁、能力库、案例库)
    │
    ├── ecom.bobqiushao.online    ← 跨境电商客群专属(=当前主页内容)
    ├── factory.bobqiushao.online ← 工厂客群专属
    ├── brand.bobqiushao.online   ← 品牌客群专属
    └── personal.bobqiushao.online ← 个人客户专属
```

### 各落地页内容差异

| 元素 | 主门户 | ecom 落地页 | factory 落地页 | brand 落地页 | personal 落地页 |
|---|---|---|---|---|---|
| Hero 主张 | "你的方案合作伙伴" | "亚马逊太贵?自建独立站" | "海外买家搜得到的工厂官网" | "多品牌矩阵运营专家" | "专业的线上名片" |
| 痛点叙事 | 通用化 | 平台佣金/风险 | 询盘/Google 收录 | 多语言/数据 | 体面/差异化 |
| 案例突出 | 全部 | 跨境案例 + Auraloop | Harrow + 工厂 demo | Auraloop 矩阵 | 个人品牌案例 |
| 价格方案 | 概览 | 跨境定价 | 工厂定价 | 矩阵定价 | 个人定价 |
| 主 CTA | 浏览能力 | 预约跨境咨询 | 预约工厂方案 | 预约矩阵咨询 | 预约个人方案 |

### 共享内容

所有落地页**共享**:
- 能力库展示(可按客群优先级排序)
- 视觉系统(同一品牌)
- 联系方式
- ADR 体现的核心方法论

---

## 备选方案

| 方案 | 优点 | 缺点 | 为什么没选 |
|---|---|---|---|
| A. 单一主页(当前) | 维护简单 | 转化率低,信息架构混乱 | 已验证不工作 |
| B. **主门户 + 子域名落地页(选定)** | 转化率高,易维护,易扩展新客群 | 初期需要重构信息架构 | 平衡了维护成本与转化率 |
| C. 4 个完全独立的网站 | 各自最优 | 维护负担 4 倍,品牌分散 | 一人公司不可承受 |
| D. 主页路径前缀(/ecom /factory) | 集中 SEO | 客户决策时不够"专属感" | 子域名的"专属感"对销售更有利 |

## 后果

### 正面
- 每个客群看到的都是"为他写的话"
- 转化率显著提升(单页 SEO 优化也更精准)
- 新客群只需新建一个子域名落地页,不影响其他
- 通过 ADR-0021 Vercel 多前缀域名方案,新增客群成本接近零

### 负面 / 已知风险
- 内容工厂工作量增加(每客群独立文案)
- 需要保持各落地页"同一品牌感",否则会显得分裂
- 风险缓解:用统一的 Design System + 共享组件库

### 触发重新评估的条件
- 当某客群营收占比 > 70% 时,可能将主门户重新偏向该客群
- 当出现完全新的客群类型时(如 SaaS / 教育),增加新落地页
- 每季度审视各落地页转化率,调整资源分配

## 实施清单

- [ ] 主门户重构:从"卖跨境方案"改为"展示能力 + 客群入口"
- [ ] 当前主页内容迁移到 `ecom.bobqiushao.online`
- [ ] 创建 `factory.bobqiushao.online` 落地页(用 Harrow demo 作为主案例)
- [ ] 创建 `personal.bobqiushao.online` 落地页(暂时简版)
- [ ] `brand.bobqiushao.online` 暂缓,等真实客户后再做
- [ ] 主门户的"客群入口"区清晰引导到对应落地页
- [ ] 配置子域名 SEO 不互相冲突(robots / canonical)
