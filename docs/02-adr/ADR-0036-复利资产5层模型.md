# ADR-0036: 复利资产 5 层模型

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Engineering Culture
**关联 ADR:** ADR-0004 (上游)

---

## 背景（Context）

独立工作者最大的陷阱是"做了很多工作，但没有积累资产"。
每次项目结束，所有的经验和工具都留在脑子里，下次还要从零开始。
Bob 需要一套系统化的资产积累模型，确保每个任务都在为长期复利做贡献。

## 决策（Decision）

将所有资产组织为 **5 层复利模型**，每一层的产出自动反哺上层：

```
Layer 5: 客户实战 (CASE)
    ↑ 产出案例，反哺能力库
Layer 4: 落地页 (LANDING)
    ↑ 产出客群洞察，反哺能力库
Layer 3: 能力库 (CAPABILITY)
    ↑ 产出可复用方案，反哺工程基础
Layer 2: 工程基础 (FOUNDATION: ADR / RUNBOOK / CI/CD)
    ↑ 产出决策记录和操作手册，反哺学习吸收
Layer 1: 学习吸收 (INPUT)
    ↑ 产出灵感和知识，反哺所有层
```

**每完成一个上层任务，自动产生下层资产：**

| 上层任务 | 自动产生的下层资产 |
|---|---|
| 完成客户项目（Layer 5） | 新案例 + 新 RUNBOOK + 新能力验证 |
| 上线落地页（Layer 4） | 客群洞察 + A/B 测试数据 |
| 扩展能力库（Layer 3） | 新 ADR + 新代码模板 |
| 更新工程基础（Layer 2） | 更高效的下次执行 |
| 完成学习整理（Layer 1） | 新灵感 + 新方法论 |

## 后果（Consequences）

### 正面
- 每个任务都有明确的"资产产出"目标
- 3 年后的资产积累远超竞争对手
- AI Agent 有明确的"什么算完成"的标准（不只是功能上线，还要有资产产出）

### 负面 / 已知风险
- 短期内每个任务的完成时间比"直接做"长
- 需要 Bob 有意识地执行资产沉淀步骤
