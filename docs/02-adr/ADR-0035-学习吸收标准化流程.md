# ADR-0035: 学习吸收的标准化流程(Inspiration → Distill → Remix)

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Engineering Culture
**关联 ADR:** 关联 ADR-0036(复利资产 5 层模型)

---

## 背景

Bob 明确表态:
> "在这个过程中我会不断地学习新技术,新案例,新 UI 素材库,新顶级网站的架构和图片作为我的内容资产。"

意味着 Bob 的工作有一半是"学习吸收",另一半是"输出执行"。

如果学习是**散乱、未沉淀**的,那么:
- 看过的顶级网站 3 个月后想不起细节
- 收藏的 UI 素材库永远在收藏夹吃灰
- 学到的新技术不会自然进入项目
- 学习的"复利效应"等于零

**长期主义的核心不只是"持续学习",更是"学习能沉淀为可调用的资产"。**

## 决策

**采用统一的"INSPIRE → DISTILL → REMIX"学习吸收流程,所有学习行为都按此流程沉淀。**

### 三阶段方法论

```
INSPIRE (吸收)
    ↓
DISTILL (提炼)
    ↓
REMIX (重组)
    ↓
OUTPUT (产出)
```

### Stage 1: INSPIRE(吸收)

**触发条件:**
- 看到一个值得借鉴的顶级网站
- 学到一个新技术 / 新工具
- 发现一个 UI 素材库
- 阅读一篇有价值的技术文章

**操作:**
1. 截图 / 录屏 / 保存原始链接
2. 30 秒之内放进 `05-capability-library/inspiration/` 目录
3. 命名:`YYYY-MM-DD-{type}-{name}.md`
4. 至少写一句话:"为什么我觉得它值得记下"

**禁止:** 收藏到浏览器书签/收藏夹(等于丢失)

### Stage 2: DISTILL(提炼)

**触发条件:**
- 周末整理一周吸收的素材
- 准备开始一个新项目时

**操作:**
1. 对 INSPIRE 的内容做特征提取
2. 按维度拆解:视觉 / 交互 / 内容架构 / 技术实现
3. 对每个维度打可借鉴度评分(1-10)
4. 写 "What to Borrow / What to Avoid"

**模板:**
```markdown
---
source_url: ...
captured_at: YYYY-MM-DD
type: website | technique | tool | ui-library
tags: [...]
distilled_at: YYYY-MM-DD
---

## Why It Caught My Attention
[一句话]

## Visual Characteristics
[详细描述]

## Interaction Patterns
[...]

## Content Architecture
[...]

## Technical Implementation
[...]

## Score
- Visual: X/10
- Interaction: X/10
- Content: X/10
- Tech: X/10

## What to Borrow
- ...

## What to Avoid
- ...

## Code Snippets (if extracted)
```
```

### Stage 3: REMIX(重组)

**触发条件:**
- 开始一个新案例
- 接到一个新客户需求
- 月度案例库更新

**核心原则:不是抄袭,是融合。**

**融合公式:**
```
案例 A 的视觉 + 案例 B 的交互 + 案例 C 的内容架构
                  ↓
         针对当前目标客群重新设计
                  ↓
              专属 REMIX 输出
```

**禁止:**
- ❌ 直接复制顶级品牌的设计或代码
- ❌ 在客户可见的地方使用顶级品牌真实名称/Logo
- ❌ 跳过 DISTILL 直接 REMIX(没有提炼,融合就是抄袭)

### Stage 4: OUTPUT(产出)

**每个 REMIX 必须最终成为:**
- ✅ 一个可访问的子域名展示(case-{name}.bobqiushao.online)
- ✅ 一份 GitHub 仓库代码
- ✅ 加入 cases.json 主门户作品集
- ✅ 一份 1 页 PDF 案例说明给客户

## 备选方案

| 方案 | 优点 | 缺点 | 为什么没选 |
|---|---|---|---|
| A. 不做沉淀,纯靠记忆 | 没有任何流程负担 | 学习资产为零,长期等于没学 | 与长期主义冲突 |
| B. 只收集不提炼 | 比记忆好 | 半年后看不懂自己存了什么 | 收集即丢失 |
| C. **完整三阶段流程(选定)** | 真正形成可调用资产 | 需要纪律 | 唯一兼顾长期复利的方案 |
| D. 用 Notion/Obsidian 等工具 | 工具完善 | 信息散落于多平台,难维护 | 倾向于 Git 单一真相源 |

## 后果

### 正面
- 1 年后形成 200+ inspiration / 50+ distill / 20+ remix 的资产
- 学习能力直接转化为业务能力
- 案例库可形成"设计模式手册",可包装为 ¥199 电子书产品
- 任何 AI Agent 接手项目都能快速理解 Bob 的设计哲学

### 负面 / 已知风险
- 需要每周 1-2 小时的整理时间
- 初期 INSPIRE → DISTILL 之间会积压
- 风险缓解:每周日固定 1 小时整理,纳入个人 RUNBOOK

### 触发重新评估的条件
- 当 inspiration 库超过 500 条时,需要分类机制升级
- 当 REMIX 节奏明显跟不上 INSPIRE 时,需要降低 INSPIRE 标准
- 每年 6 月审视:这个流程是不是还在工作
