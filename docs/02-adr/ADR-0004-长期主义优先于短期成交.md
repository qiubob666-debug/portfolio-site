# ADR-0004: 长期主义优先于短期成交

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Strategic
**关联 ADR:** ADR-0001 (上游) / ADR-0010 (下游)

---

## 背景（Context）

独立工作者面临的最大陷阱是"为了快速成交而牺牲可复用性"。
每次为了一个客户做定制化但不可复用的工作，都是在消耗时间而非积累资产。
Bob 的工作模式（个人 + AI Agent）需要一套明确的决策优先级，避免在"快速成交"和"资产积累"之间反复摇摆。

## 决策（Decision）

任何"快速成交 vs 可复用资产"的二选一，默认选**可复用资产**。

## 备选方案（Alternatives Considered）

| 方案 | 优点 | 缺点 | 为什么没选 |
|---|---|---|---|
| 短期成交优先 | 现金流快 | 无资产积累，重复劳动 | 3 年后与今天无差别 |
| 长期主义优先（本方案） | 资产复利增长 | 短期收入可能较慢 | 已接受 |
| 混合策略（无原则） | 灵活 | 决策成本高，AI Agent 无法遵循 | 缺乏可操作的规则 |

## 后果（Consequences）

### 正面
- 每个任务都产出可复用资产（ADR / RUNBOOK / 代码模板 / 案例）
- AI Agent 有明确的优先级规则，减少决策摩擦
- 3 年后的能力库 + 案例库 = 真正的护城河

### 负面 / 已知风险
- 短期内可能因为"太慢"而失去部分冲动型客户
- 需要 Bob 有足够的现金流储备支撑

### 触发重新评估的条件
- 当账上现金 < 3 个月生活费时，可临时切换为成交优先
- 每季度末复盘：资产积累速度是否符合预期

## 实施清单（Implementation）

- [x] 所有 P0/P1 任务设计标准包含"是否产出可复用资产"
- [x] ADR / RUNBOOK 体系作为资产积累的基础设施
- [ ] 建立季度复盘机制，追踪资产积累速度
