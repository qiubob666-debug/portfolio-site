# ADR-0041: 客户数据与凭据保护规范

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Legal & Compliance
**关联 ADR:** ADR-0034 (关联)

---

## 背景（Context）

Bob 在为客户工作时，会接触到客户的域名凭据、Hosting 账号、数据库密码等敏感信息。
一旦这些信息泄露（如意外写入 Git），后果极其严重（客户数据泄露、法律责任）。
需要一套严格的凭据保护规范，保护客户和 Bob 自身。

## 决策（Decision）

**凭据存储规范：**

| 凭据类型 | 存储方式 | 禁止方式 |
|---|---|---|
| 域名 / Hosting 账号 | 1Password / Bitwarden | 写入 Git / 明文文档 |
| 数据库密码 | GitHub Secrets / Vercel Env | 写入代码 / 写入 Git |
| API Keys | GitHub Secrets / Vercel Env | 写入代码 / 写入 Git |
| 客户 FTP/SSH | 1Password / Bitwarden | 写入 Git / 明文邮件 |

**AI Agent 处理凭据规范：**
- AI Agent 处理客户凭据时，**全程禁止打印明文**
- AI Agent 不能将凭据写入任何文件（包括临时文件）
- AI Agent 只能通过环境变量读取凭据

**凭据接收流程：**
1. 客户通过加密渠道（1Password 共享 / 加密邮件）发送凭据
2. Bob 存入 1Password / Bitwarden
3. 需要时通过 GitHub Secrets / Vercel Env 注入到 CI/CD

**项目结束凭据处理：**
- 项目结束后，客户可申请凭据销毁
- Bob 从 1Password 删除客户凭据
- 通知客户修改相关密码

**`.gitignore` 必须包含：**
```
.env
.env.local
.env.*.local
*.pem
*.key
secrets/
credentials/
```

## 后果（Consequences）

### 正面
- 客户数据安全有制度保障
- Bob 自身法律风险大幅降低
- 建立专业形象（客户感受到安全感）

### 负面 / 已知风险
- 凭据管理增加了一定的操作摩擦
- 需要 Bob 养成使用密码管理器的习惯
