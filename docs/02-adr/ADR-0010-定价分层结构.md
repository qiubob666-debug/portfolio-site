# ADR-0010: 定价分层结构

**日期:** 2026-04-23
**状态:** Accepted
**决策者:** Bob Qiushao
**层级:** Commercial
**关联 ADR:** ADR-0001 (上游) / ADR-0012 (下游) / ADR-0013 (下游)

---

## 背景（Context）

原有定价（¥3,800 / ¥8,800 / ¥18,000）过低，与 Bob 实际能力和市场定位不符，且无法覆盖合理的时间成本。
需要重新建立价格体系，同时引入订阅类产品作为稳定现金流来源。

## 决策（Decision）

采用 3 档主产品 + 3 类订阅产品的定价结构，全面提升价格锚点。

| 档位 | 价格 | 适用场景 | 交付时长 |
|---|---|---|---|
| 基础 | ¥8,800 起 | 单一能力交付（如展示站） | 10 天起 |
| 推荐 | ¥22,800 起 | 能力组合（如电商 + Auth） | 28 天起 |
| 高端 | ¥58,000 起 | 矩阵 / 多门户 / 完整方案 | 60 天起 |
| 月度运维 | ¥1,980/月起 | 持续服务订阅 | 持续 |
| 年度增长包 | ¥28,800/年起 | 含内容工厂自动化 | 持续 |
| 能力加购 | ¥9,800 起 | 单项能力（如 Auth）加购 | 按需 |

## 备选方案（Alternatives Considered）

| 方案 | 优点 | 缺点 | 为什么没选 |
|---|---|---|---|
| 维持旧价格 | 成交门槛低 | 无法覆盖成本，定位混乱 | 与能力不匹配 |
| 更高价格（¥100k+） | 高客单价 | 当前案例库不足以支撑 | 时机未到 |
| 新定价（本方案） | 与市场真实对手对齐 | 需要重新培育客户认知 | 已接受 |

## 后果（Consequences）

### 正面
- 价格与 Bob 真实能力和市场对手对齐
- 月度订阅模型提供稳定现金流基础
- 交付时长延长（7→10 天）反而提升专业感

### 负面 / 已知风险
- 价格提升后，短期成交数量可能下降
- 需要更强的案例支撑来说服高价值客户

### 触发重新评估的条件
- 第 5 个客户成交后，根据真实数据校准各档位
- 当月度运维客户超过 10 个时，考虑提升订阅价格
