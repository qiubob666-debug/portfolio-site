**日期:** 2026-04-23

## 1. 背景与目标

根据 `ADR-0001`，`bobqiushao.online` 定位为**个人能力的总展示中心**，而非单一产品销售页。结合 `ADR-0036` 的复利资产 5 层模型，个人作品集（Layer 5）是反哺能力库（Layer 3）和落地页（Layer 4）的核心资产。

当前项目技术栈为 **Vite + React + TypeScript + TailwindCSS + Framer Motion + Radix UI**。为了提升作品集的视觉表现力和交互体验，本方案搜集了 5 个业界顶尖的开发者个人网站作为设计参考，并推荐了与当前技术栈高度契合的开源动画组件库，最后结合现有的 ADR 和 Runbook 提出整合建议。

## 2. 顶尖个人网站作品集参考 (Top 5)

以下 5 个案例代表了当前前端开发与设计工程（Design Engineering）的最高水准，其设计理念与 `3c-cases-ux-strategy.md` 中强调的“交互式叙事”高度一致。

| 案例名称 | 网址 | 核心技术栈 | 设计特点与借鉴价值 |
|---|---|---|---|
| **Brittany Chiang (v4)** | [brittanychiang.com](https://brittanychiang.com/) | Gatsby, Styled Components | 经典的深色主题（Navy/Slate 配色），极具极客感。侧边栏固定导航与右侧滚动内容的经典布局。鼠标悬停时的探照灯（Flashlight）效果引导视觉焦点。经历与项目采用卡片式布局，微交互极其细腻。适合用于展示 `resume/README.md` 中的工作经历和核心项目（如 AuraLoop 生态系统）[1]。 |

## 3. 推荐开源 UI 组件库

为了在 Vite + React + Tailwind 栈中快速实现上述案例级别的视觉效果，推荐引入以下现代动画组件库。这些库大多基于 **Framer Motion** 和 **Tailwind CSS**，支持 Copy-Paste 模式，无需安装臃肿的 npm 包。

| 组件库名称 | 核心特点 | 适用场景与整合建议 |
|---|---|---|
| **Magic UI** | 专为 Design Engineers 打造，提供 150+ 免费开源的动画组件[6]。 | **Bento Grid (便当盒网格):** 用于展示 3C 案例（如 LUMINOS 手机特性）。<br>**Text Reveal (文字渐显):** 用于 Hero 区的 Slogan 展示。<br>**Globe / Particles:** 用于展示全球化部署或 CDN 架构（对应 `ADR-0026`）。<br>**Marquee (跑马灯):** 用于展示技术栈 Logo 或合作品牌。 |
| **Aceternity UI** | 提供 200+ 生产级动画组件，包含大量炫酷的 3D 和光影效果[7]。 | **Glare Card (眩光卡片):** 适合用于展示高价值的“能力组合”（如 Auth+Comm 组合）。<br>**Parallax Scroll (视差滚动):** 完美契合 `3c-cases-ux-strategy.md` 中 SKYVEX 无人机案例的全屏沉浸需求。<br>**Tracing Beam (光束追踪):** 适合用于连接架构图中的不同节点（如展示三层架构 PGC/Commerce/UGC 的数据流向）。 |
| **React Bits** | 专注于文本、背景和 UI 元素的创意动画集合[8]。 | **Animated Backgrounds:** 为不同的客群落地页（Layer 4）提供差异化的背景氛围。<br>**Text Animations:** 用于增强页面标题的视觉冲击力。 |

## 4. 整合执行建议 (Runbook 扩展)

结合 `RUNBOOK-10-add-new-case.md` 和 `ADR-0040`（案例展示标注规范），建议在个人门户的开发中采取以下整合步骤：

### 4.1 架构层整合

在 `portfolio-site` 中创建 `components/magicui` 和 `components/aceternity` 目录，采用 Copy-Paste 方式引入所需组件，避免过度依赖外部包。在 `tailwind.config.js` 中统一定义动画曲线（如借鉴 Emil Kowalski 的 spring 配置），确保全站动效一致。

### 4.2 内容层整合

参考 Brittany Chiang 和 Paco Coursey，使用极简排版结合 Magic UI 的 Text Reveal 动画重构 Hero 区，突出 "Frontend Engineer & Automation Expert" 定位。

对于能力库展示 (Capability Library)，参考 Lee Robinson 的列表设计，将 `ADR-0001` 中定义的 4 大能力（Auth+Comm, Matrix, Content, Frontend）以 Aceternity UI 的 Hover Effect 卡片展示。点击卡片展开详细的 Tech Stack 和 Pricing 信息。

对于案例库展示 (Cases Section)，严格执行 `ADR-0040`，使用不同颜色的 Badge 区分 `Live` (如 Auraloop), `Concept` (如 3C 案例), `Demo`。使用 Magic UI 的 Bento Grid 布局展示案例截图和标签。

对于简历与经历 (Resume)，将 `resume/README.md` 中的内容转化为交互式的时间轴（Timeline），参考 Brittany Chiang 的侧边栏导航模式。

### 4.3 自动化与复利 (Layer 2 -> Layer 3)

更新 `RUNBOOK-10`：在 "Step 3: 项目结构标准化" 中，要求新案例必须使用至少一种高级交互组件（如 Parallax 或 Glare Card），以持续丰富个人的前端能力库。
